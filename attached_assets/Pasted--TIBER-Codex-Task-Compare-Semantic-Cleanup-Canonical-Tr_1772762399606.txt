## Primary objectives

### Objective A — Clean up compare semantics

Tighten the canonical comparison output so labels, pillar meanings, and reasons stop contradicting themselves.

Fix the following classes of issues:

1. **Verdict label consistency**
   - Human-readable `verdict.label` must align with machine `edge_strength`
   - Example:
     - `edge_strength = strong` should not produce `Lean X`
     - `Lean X` should generally map to `slight` or `moderate`, depending on current system conventions
   - Preserve current verdict taxonomy structure:
     - `winner`
     - `edge_strength`
     - `actionability`
     - `label`

2. **Pillar semantic consistency**
   - Supporting notes and metrics must actually fit the pillar they justify
   - Do not use snap share notes as primary support for `efficiency` unless the service explicitly defines that and documents it
   - If existing underlying compare logic is coarse, use mapper-layer discipline to avoid obviously wrong pillar explanations

3. **Reason wording consistency**
   - Reasons must match the actual metrics included in `evidence.metrics`
   - If metric source is pass EPA allowed, do not describe it as run defense
   - Remove wording drift and category confusion

4. **Summary signal improvement**
   - Expand `evidence.summary_signal` so it contains more than a thin `market_delta`
   - Without adding new football logic, expose clearer top-level comparison primitives if already available from existing comparison output
   - Good examples:
     - `side_a_score`
     - `side_b_score`
     - `score_delta`
     - existing market delta if available
